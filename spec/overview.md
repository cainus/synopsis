# Synopsis — Overview

Synopsis is a desktop app for inspecting the changes on a git branch before merging. You point it at any local git repository and it gives you four views of your branch: what files changed, what the changes mean, what tests changed and how, and a visual impact graph of the before/after structure.

## Problem

Before opening a pull request it's useful to have a clear picture of what your branch actually contains — but `git log` and `git diff` are low-level tools that require mental assembly. Synopsis does that assembly for you.

## Core flow

1. Launch the app
2. Click **Choose Repo Folder** and select a directory containing a git repo
3. The app detects the default branch and loads the **Summary** and **Delta** tabs automatically
4. Switch to **Tests** or **Diagrams** to trigger those analyses on demand

## Tabs

| Tab | What it shows | When it loads |
|-----|---------------|---------------|
| Summary | A brief headline and paragraph summary of the changes | Eagerly on folder pick (default tab) |
| Details | Hierarchical breakdown of product and technical changes with code snippets | On first visit to the tab |
| Delta | Files changed vs the default branch, with line counts and status colours | Eagerly on folder pick |
| Tests | Each changed test case with a description of the behaviour change | On first visit to the tab |
| Diagrams | Before/after Mermaid impact graphs of the changed functions/methods, generated by Claude | On first visit to the tab |

Summary is the first tab in the tab bar and the default active tab. Details is the second tab and uses a separate Claude call from Summary.

## Diff modal

Clicking on a file (Delta tab), a code snippet link (Summary tab), or a test case (Tests tab) opens a shared diff modal that shows the relevant code changes with colour-coded add/remove lines. The modal supports both inline and side-by-side views.

## Refresh

A **↺** button appears next to the folder path once a repo is selected. Clicking it clears all loaded data and re-fetches from scratch (useful after making more commits).
