# Synopsis — Overview

Synopsis is a desktop app for inspecting the changes on a git branch before merging. You point it at any local git repository and it gives you three views of your branch: what files changed, what the changes mean, and what tests changed and how.

## Problem

Before opening a pull request it's useful to have a clear picture of what your branch actually contains — but `git log` and `git diff` are low-level tools that require mental assembly. Synopsis does that assembly for you.

## Core flow

1. Launch the app
2. Click **Choose Repo Folder** and select a directory containing a git repo
3. The app detects the default branch and loads the **Delta** tab automatically
4. Switch to **Summary** or **Tests** to trigger those analyses on demand

## Tabs

| Tab | What it shows | When it loads |
|-----|---------------|---------------|
| Delta | Files changed vs the default branch, with line counts | Immediately on folder pick |
| Summary | A narrative description of the changes, generated by Claude | On first visit to the tab |
| Tests | Each changed test case with a description of the behaviour change | On first visit to the tab |

## Refresh

A **↺** button appears next to the folder path once a repo is selected. Clicking it clears all loaded data and re-fetches from scratch (useful after making more commits).
